1. Дане оновлення потребує декількох файлів: 
   - 1.1. actual_labor_tariff.xlsx (зміна з назвою цього файлу у скрипті - file_path_tariff)- це файл із ставками зарплат по посадах. Оновлювати цей файл потрібно двічі - у квітні та жовтні.
У скрипті labor.py визначена назва колонк у цьому файлі: actual_labor_tariff_columns = [
    "№ магазина", "адреса",	"посада	вартість години без податків з 04_26", "вартість години з податками з 04_26"
] - тож при зміні файлу треба в скрипті оновити назву колонок на актуальні.
   - 1.2. Посади.xlsx - словник посад у відділах, якщо з'являться нові посади, то просто додати їх у цей файл
   - 1.3. labor_yymm.xlsx - файл з даними для актуального місяця. У файлі conn.py визначені назви колонок для цього файлу: labor_columns = [
    "filid", "mm", "h",	"region", "filialname"
] - переконайся, що у файлі будуть саме такі назви колонок (за потреби змінити їх треба у файлі prod_columns_archive.py)
2. Відкрити скрипт labor.py, знайти на початку змінну file_path - ця змінна каже програмі, з якого джерела брати дані. Змінити назву на актуальну (тобто на поточний місяць).
Приклад file_path = "labor_2605.xlsx" - тут треба виправити рік-місяць, якщо зараз 06.2026, то коректна назва файлу буде "labor_2606.xlsx". Переконайся, що файл з такою назвою є в папці labor.
